|------|------|

## 快速烧录

### 1. 配置密钥

```bash
cp main/mimi_secrets.h.example main/mimi_secrets.h
# 编辑 mimi_secrets.h，填入 Wi-Fi SSID/密码、API Key 等
```

修改 `mimi_secrets.h` 后必须 **fullclean** 再编译：

```bash
idf.py fullclean && idf.py build
```

### 2. 编译

```bash
# 确保在项目根目录
idf.py build
```

或直接：

```bash
idf.py set-target esp32s3
idf.py build
```

### 3. 烧录

**标准烧录：**

```bash
idf.py -p /dev/ttyUSB0 flash
```

**如果烧录失败（压缩模式问题）：**

ESP32-S3 有已知的 flash 压缩 bug，错误信息：
```
A fatal error occurred: Failed to leave compressed flash mode
```

此时改用 esptool 直连并关闭压缩：

```bash
# 先杀掉占用串口的进程
fuser -k /dev/ttyUSB0 2>/dev/null
sleep 1

# 用 esptool 直接烧录（--no-compress 是关键）
python -m esptool --chip esp32s3 -p /dev/ttyUSB0 -b 460800 \
  --before default-reset --after hard-reset write-flash \
  --flash-mode dio --flash-size 16MB --flash-freq 80m \
  --no-compress \
  0x0          build/bootloader/bootloader.bin \
  0x8000       build/partition_table/partition-table.bin \
  0xf000       build/ota_data_initial.bin \
  0x20000      build/mimiclaw.bin \
  0x420000     build/spiffs.bin
```

> **`--no-compress`** 参数是 ESP32-S3 烧录的关键。如果 `idf.py flash` 失败，始终用这个方法重试。

### 4. 打开串口监视器

```bash
idf.py -p /dev/ttyUSB0 monitor
```

退出监视器：`Ctrl+]`

## 两个 USB 端口说明

ESP32-S3 开发板有两个 USB 端口，连接到电脑后显示的设备名不同：

| 端口 | 用途 | 设备节点 |
|------|------|----------|
| USB (JTAG) | 烧录 + 监视器 | `/dev/ttyACM0` |
| COM (UART) | 交互式 CLI | `/dev/ttyUSB0` |

先确认实际端口：

```bash
ls /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
```

通常插上电脑后，烧录和日志用 `/dev/ttyACM0` 即可。如果 `/dev/ttyACM0` 连不上再试 `/dev/ttyUSB0`。

## SPIFFS 分区说明

固件包含一个 12MB 的 SPIFFS 分区，存放运行时数据：

| 路径 | 用途 |
|------|------|
| `/spiffs/config/SOUL.md` | AI 人格设定 |
| `/spiffs/config/USER.md` | 用户信息 |
| `/spiffs/memory/MEMORY.md` | 长期记忆 |
| `/spiffs/memory/YYYY-MM-DD.md` | 每日笔记 |
| `/spiffs/sessions/*.jsonl` | 对话历史 |
| `/spiffs/cron.json` | 定时任务 |
| `/spiffs/HEARTBEAT.md` | 心跳任务 |
| `/spiffs/skills/*.md` | 自定义技能 |

SPIFFS 数据源在 `spiffs_data/` 目录，编译时自动打包。

## 分区表（16MB Flash）

| 偏移 | 大小 | 名称 | 用途 |
|------|------|------|------|
| 0x9000 | 24KB | nvs | ESP-IDF 内部 + 应用配置 |
| 0xF000 | 8KB | otadata | OTA 启动状态 |
| 0x20000 | 2MB | ota_0 | 固件槽 A |
| 0x220000 | 2MB | ota_1 | 固件槽 B |
| 0x420000 | 12MB | spiffs | 存储分区 |
| 0xFF0000 | 64KB | coredump | 崩溃转储 |

## 常见问题

### Q: `idf.py build` 找不到 `json` 组件？

ESP-IDF v6.x 移除了 `json` 组件。项目已改用上游 `cJSON v1.7.15`（`main/cJSON_upstream.c/h`）。如果遇到此错误，检查 `main/CMakeLists.txt` 的 REQUIRES 中不应包含 `json`。

### Q: 烧录后串口没有输出？

- 确认串口是 `/dev/ttyUSB0`
- 检查是否有其他程序占用了串口（`fuser -k /dev/ttyUSB0`）
- 尝试重新拔插 USB 线
- 按住 Boot 键 + 按 Reset 键 + 松开 Boot 键，进入下载模式

### Q: 如何只烧录固件不烧录 SPIFFS？

去掉 `0x420000 build/spiffs.bin` 部分即可。首次烧录或 SPIFFS 结构变化时必须烧录 SPIFFS。

### Q: 烧录过程中 `A fatal error occurred: Could not open /dev/ttyUSB0`？

串口被占用了。杀掉占用进程：

```bash
fuser -k /dev/ttyUSB0
sleep 1
# 重新烧录
```
